# 自动化 Agent 执行契约

本文件定义所有自动化 agent 必须遵守的最低执行规则。目标是让不同 agent 执行同一个 workflow 时产生一致、可追踪、可恢复的结果。

## 适用范围

凡是执行公司自动化 workflow 的 agent，都必须遵守本契约。

包括但不限于：

1. 需求解析。
2. 登录系统。
3. 页面筛选。
4. 文件导出。
5. Excel 处理。
6. 任务创建。
7. 数据导入。
8. 报表生成。

## 总原则

稳定优先于聪明。可复核优先于一步到位。能恢复优先于重跑。

## 强制规则

### 规则 1：执行 workflow 前必须创建运行目录

任何 workflow 开始前，必须先调用或遵守 `checkpoint-runner` 的运行目录规范。

不允许把文件散落在当前目录、下载目录或临时目录中。

标准目录：

```text
runs/YYYY-MM-DD/流程英文ID-城市-批次/
  input/
  state/
  outputs/
  evidence/
  logs/
```

### 规则 2：必须写 `state/run_state.json`

每个 workflow 必须有状态文件。每个步骤开始、完成、失败、等待人工确认时都要更新状态。

没有状态文件时，不允许继续执行真实业务动作。

### 规则 3：真实业务动作必须先检查 dry-run

第一阶段 `dry_run` 必须为 `true`。

当 `dry_run=true` 时，禁止：

1. 登录真实系统。
2. 下载真实业务文件。
3. 上传文件。
4. 导入客户数据。
5. 创建外呼任务。
6. 修改飞书在线表格。

### 规则 4：高风险动作必须人工确认

以下动作即使未来进入真实执行阶段，也必须默认人工确认：

1. 创建火山外呼任务。
2. 上传或导入迈鲸客户/商机数据。
3. 修改飞书在线表格。
4. 退款。
5. 转移门店。
6. 批量下发运维任务。

### 规则 5：错误必须中文可读

错误信息必须包含：

1. 失败步骤中文名。
2. 失败原因。
3. 建议查看的证据文件。
4. 可恢复点。

示例：

```text
步骤失败：订客多日期筛选
失败原因：导出文件中存在非目标日期数据
建议查看：evidence/screenshots/date-filter-after.png
可恢复点：dingkeduo_filter_date
```

### 规则 6：浏览器操作不得依赖坐标

必须优先使用：

1. 页面文字。
2. label。
3. role。
4. placeholder。
5. DOM 属性。
6. 接口请求。

坐标点击只能作为临时人工排查手段，不允许写入正式 skill。

### 规则 7：外部系统动作必须留证据

任何外部系统动作都必须保存证据：

| 动作 | 证据 |
|---|---|
| 登录 | 登录前后截图或接口响应 |
| 筛选 | 筛选条件截图、页面状态 JSON |
| 导出 | 下载文件、导出条件、文件校验报告 |
| 上传 | 上传前确认清单、上传回执 |
| 创建任务 | 任务配置快照、任务 ID 或页面回执 |

### 规则 8：需求模糊时必须停下确认

当飞书需求中缺少城市、区县、品类、日期、人员、目标文件或关键业务条件时，必须生成确认问题，不能猜测执行。

### 规则 9：复用已有 skill，不能在 workflow 里重复实现通用能力

workflow 只做编排，不直接写复杂登录、Excel、页面定位、状态管理逻辑。

必须优先调用：

1. `checkpoint-runner`
2. `feishu-requirement-parser`
3. `excel-transform`
4. `browser-helpers`
5. 对应登录 skill

### 规则 10：完成后必须写明下一步

每次执行结束时，必须说明：

1. 当前停在哪一步。
2. 产物在哪里。
3. 是否需要人工确认。
4. 下次从哪个步骤继续。

## 新 workflow 必须包含的章节

每个 workflow 的 `SKILL.md` 必须包含：

1. 当前阶段。
2. 输入。
3. 输出。
4. 必须先调用的 skill。
5. 运行目录规范。
6. 步骤状态机。
7. 人工确认点。
8. dry-run 禁止动作。
9. 失败定位。
10. 可恢复点。

## 判断标准

一个 workflow 是否合格，不看它是否一次跑完，而看：

1. 换一个 agent 是否能按同样规则执行。
2. 失败后是否能定位问题。
3. 失败后是否能从当前步骤继续。
4. 人工是否能快速复核它做了什么。
5. 高风险动作是否没有越过确认。
